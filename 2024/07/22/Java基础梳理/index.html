一、Java集合

1.1 Collection接口

  • List
    • ArrayList：底层结构为数组，可以动态扩容
    • LinkedList：底层结构为链表，内存空间不连续，因此不支持快速随机访问
      • ArrayList&&LinkedList：ArrayList由数组实现，随机访问快，但插入删除效率较低；LinkedList相反
    • Vector：List的古老实现，线程安全
  • Set
    • HashSet：底层就是HashMap，key存数值，value存固定值。元素无序且唯一
    • LinkedHashSet：底层就是LinkedHashMap，元素有序且唯一
    • TreeSet：底层采用红黑树实现，元素有序且唯一
  • Queue
    • PriorityQueue：优先队列，堆实现
    • DelayQueue：延迟队列，队列中元素需实现Delayed接口，且按过期时间排序。越快过期的元素越靠队头；支持阻塞获取元素
    • ArrayDeque：双端队列，两端都允许插入弹出

1.2 Map接口

  • Map
    • HashMap：数组+链表/红黑树
    • LinkedHashMap：HashMap+双向链表，在HashMap的基础上维护了键值对的顺序
    • HashTable：数组+链表
      • HashMap&&HashTable：
        1. HashMap线程不安全，HashTable线程安全
        2. HashMap效率高，HashTable效率低
        3. HashMap可接受NULL键值，HashTable不可接受
        4. HashMap继承AbstractMap，HashTable继承Dictionary
        5. HashMap底层数组初始容量16，每次扩容容量为2n，若指定容量，则初始容量会是2的幂；HashTable的是11，每次扩容容量为2n+1，若指定容量，初始容量就是指定的
        6. HashMap对哈希值高位低位进行了混合扰动以减少冲突，而HashTable直接使用键的hashCode()值
    • TreeMap：红黑树

1.3 零碎知识点

1.3.1 ArrayList扩容机制

扩容可分为两种情况：

  • 当ArrayList的容量为0时，此时添加元素的话，需要扩容，三种构造方法创建的ArrayList在扩容时略有不同：

    1. 无参构造，创建ArrayList后容量为0，添加第一个元素后，容量变为10，此后若需要扩容，则正常扩容
    2. 传容量构造，当参数为0时，创建ArrayList后容量为0，添加第一个元素后，容量为1，此时ArrayList是满的，下次添加元素时需正常扩容。
    3. 传列表构造，当列表为空时，创建ArrayList后容量为0，添加第一个元素后，容量为1，此时ArrayList是满的，下次添加元素时需正常扩容

  • 当ArrayList的容量大于0，并且ArrayList是满的时，此时添加元素的话，进行正常扩容，每次扩容到原来的1.5倍

1.3.2 Comparable和Comparator

  1. Comparable表示“能够比较”，一般自定义的，需要实现比较的类会实现该接口，在自定义类内部实现比较具体逻辑；Comparator是一个比较器，用于在外部排序方法对某个类进行比较
  2. 实现Comparable需要实现compareTo方法；实现Comparator需要实现compare方法

1.3.6 为何HashMap数组长度一定要2的n次方？

因为hash%length == hash&(length-1)的前提是 length 是 2 的 n 次方，length为2的n次方之后，取余操作就可用位运算代替，速度更快

原理：假设length=2^n，那么hash/length等价于hash除以n个2，也就是将hash向右移动n位，所以取余的结果就是被移掉的n位，也就是hash的低n位，hash&(length-1)就是计算这n位的

1.3.7 HashMap的线程不安全

  • 死循环

    JDK1.7及以前，由于链表采用头插法，多线程环境下可能导致指针错误，导致死循环；但JDK1.8及以后改为尾插法，问题不存在

  • 数据覆盖

    这个是一直有的问题，一个线程可能把另一个线程的数据覆盖
