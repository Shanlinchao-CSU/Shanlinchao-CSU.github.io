腾讯金融科技

Java后端开发（一面）

  1. 常见的I/O模型了解吗？

    • BIO：

      追问：说说常见的BIO模型的应用（Reader,Writer,InputStream,OutputStream等都是BIO的实现）

    • NIO：

      Java NIO使用了多路复用技术，提供了三个组件：Channel，Buffer和Selector

      Channel就是通道，通道连接着客户端和服务端，用来传递数据，因此通常一个通道表示一个连接；Buffer表示缓冲区，用户从Channel中读数据，或往里面写数据都要通过Buffer实现。传统BIO基于字节流，读写都必须按顺序进行，但是Buffer提供了一系列方法，使用户可以自行移动读写下标；Selector是选择器，当检测到Channel事件时（Java NIO只定义了四个事件：SelectionKey.OP_CONNECT，SelectionKey.OP_ACCEPT，SelectionKey.OP_READ，SelectionKey.OP_WRITE），对这个Channel进行处理

    • AIO：

      基于事件回调机制，应用程序发起 I/O 操作后，内核负责数据传输过程，完成后通知应用程序。应用程序无需等待数据传输，可执行其他任务

  2. 同步/异步 和 阻塞/非阻塞 的区别

    同步/异步强调发出请求后，是否需要等待执行结果才能执行后续操作

    阻塞/非阻塞强调等待消息通知时的状态，阻塞表示调用结果返回之前当前线程被挂起，而非阻塞则得到结果之前不会阻塞

  3. 多路复用的“多路”指的是什么？

    指同时监听的多个I/O事件

  4. Netty具体封装了哪些复杂的操作？

    Netty主要对NIO基本的Channel，Selector和Buffer进行了更高层的封装，屏蔽了底层的复杂性

    Netty提供了对多种协议的很好支持，封装了对多协议支持的复杂性

    Netty封装了三种Reactor模型：单线程单Reactor，多线程单Reactor，多线程多Reactor

  5. 谈谈你对ZooKeeper的理解

    保证分布式数据一致性的解决方案，可用于统一配置管理、命名服务，分布式锁，集群服务等。

    zk保证了一致性和分区容错性，牺牲了部分可用性

  6. 说明Redis的选举机制，解释为什么是过半投票

    过半机制为了防止脑裂

  7. Redis集群怎么确定主节点挂了？如何选举新节点？如何保证最新数据？

    确定主节点宕机：定期PING，若不通，则询问其它哨兵，当过半数哨兵认为宕机，则通过Raft算法选出领头哨兵进行故障恢复

Java后端开发（一面A）

  1. 进程通信方式

    • 管道：半双工，有特定的读端和写端

      • 无名管道(pipe)：只允许有亲缘关系的进程之间使用

      • 命名管道(fifo)：允许任何进程之间使用

    • 消息队列：一个消息的链表，是一系列保存在内核中消息的列表。用户进程可以向消息队列添加消息，也可以读取消息。消息队列与管道通信相比，其优势是对每个消息指定特定的消息类型，接收的时候不需要按照队列次序，而是可以根据自定义条件接收特定类型的消息

    • 共享内存：效率高

    • 信号量
