Java后端开发（一面）

流程：

1. 自我介绍

2. 讲解项目

+ 这里因为我第一个RPC项目复习的不是很充分，整体讲了非常久，且讲解过程中逻辑混乱，条理不清晰。虽然说该提到的要点都提到了，但是被面试官指出**需要精简，否则面试官很难提取到介绍中的重点**

3. 提问

    1. 介绍一下Set和List，说明它们的区别

    2. 介绍一下HashMap和HashTable的区别（没回答出来，没有接触过HashTable）

    |            |      HashMap       |       HashTable        |
    | :--------: | :----------------: | :--------------------: |
    |  线程安全  |         否         |           是           |
    |    效率    |         高         |           低           |
    |    实现    |  继承AbstractMap   |     继承Dictionary     |
    |    NULL    | 可以用NULL作为键值 | 不可以使用NULL作为键值 |
    |    扩容    |    初始容量为16    |      初始容量为11      |
    | 迭代器容错 |      快速失败      |       非快速失败       |

    3. HashMap有序吗？在HashMap的子类中，哪个是有序的？

    无序，LinkedHashMap

    4. 分别介绍一下List和Map的遍历方式

    5. Java实现多线程的方式（只知道Thread和线程池）

    + **继承**Thread类，调用start()方法
    + **实现**Runnable接口，并实例化之来构造Thread实例，在通过Thread实例的start()方法
    + 继承Callable接口
    + 线程池

    6. 解释下线程进程区别（只回答了进程大线程小，一个进程拥有多个线程）

    |          | 进程                                                         | 线程                                                         |
    | -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
    | 根本区别 | 操作系统资源分配基本单位                                     | 处理器任务调度和执行的基本单位                               |
    | 资源开销 | 每个进程都有独立的代码和数据空间，程序之间的切换会有较大的开销 | 同一类线程共享代码和数据空间，每个线程都有自己独立的运行栈和程序计数器，线程之间切换的开销小 |
    | 包含关系 | 一个进程可以包含多个线程                                     | /                                                            |
    | 内存分配 | 地址空间和资源相互独立                                       | 同进程的线程共享本进程的地址空间和资源                       |
    | 影响关系 | 崩溃后，在保护模式下不会对其它进程影响                       | 崩溃后，整个进程都崩溃                                       |
    | 执行过程 | 每个独立的进程有程序运行的入口、顺序执行序列和程序出口       | 线程不能独立执行，必须依存在应用程序中，由应用程序提供多个线程执行控制，两者均可并发执行 |

    7. 用过什么数据库（MySQL，ORACLE，SQLServer，Redis，MongoDB），简单介绍下Redis

    8. 知道哪些排序算法（冒泡，堆，桶，快排，分治），选一个解释一下

    9. SpringBoot的拦截器用过吗？具体怎么实现？（实在记不得具体实现了）

总结：难度不大，时间较短，但自己没有准备充分，有点紧张，面试感觉不好
